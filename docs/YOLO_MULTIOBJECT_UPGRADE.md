# YOLO 多目标检测与自动裁剪升级技术文档

## 1. 升级背景
原系统采用相机位（F0-F8）与仪器类型的强绑定模式，导致一个相机只能对应一台仪器的局限性。随着实验室布局的变化，单张全景大图中往往包含多个仪表。为此，我们引入了基于 **YOLO** 的多目标检测方案，实现了“全图扫描、自动抠图、精准读取”的闭环。

**注意：以后仪器统一称为 D0-D8；相机统一称为 F0-F8。**

## 2. 核心技术组件

### 2.1 YOLO 引擎
- **模型版本**：YOLOv8 Custom。
- **分类定义**：
  - 类别 0-8：分别对应 D0-D8 仪器识别模板。
- **手动 NMS 算法**：
  - **解决方案**：在 `yolo_detector.py` 中引入 NMS，设置 `iou_threshold=0.15` 和 `agnostic_nms=True`（跨类别抑制）。
  - **效果**：解决了同一个仪表被画出多个重叠框的问题。

### 2.2 自动裁剪与存储流 (Auto-Crop)
- **逻辑实现**：在 `instrument_reader.py` 中，YOLO 检测到的每个 bbox 都会被增加 15px 的 Padding 后进行裁剪。
- **存储路径**：裁剪图自动保存至 `camera_images/F{camera_id}/crops/` 目录下。
- **命名规范**：`原图名_crop_D{instrument_id}_时间戳.jpg`，确保唯一性。

### 2.3 相机与仪器解耦架构
- **API 层面**：`/experiments/{exp_id}/run-test` 接口不再盲目读取全图。它会调用 YOLO 获取所有目标，并智能寻找 `D{id}` 最佳裁剪图。
- **前端展示**：系统会将数据库中该条读数关联的图片路径更新为**裁剪后的特写图**。用户在前端看到的将是精准的仪表特写（D0~D8）。
