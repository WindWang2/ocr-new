# 仪器读数识别系统

使用本地部署 多模态模型进行高精度仪器读数识别。

## 命名规范 (Naming Convention)

为了区分逻辑，系统采用以下命名规范：
- **仪器 (Instruments)**: 命名为 **D0, D1, ..., D8**。
- **物理相机 (Cameras)**: 命名为 **F0, F1, ..., F8**。

目前 9 类仪器的标准映射关系如下：
| 仪器 ID | 名称 | YOLO 类别 | 物理相机 |
| :--- | :--- | :--- | :--- |
| **D0** | 吴英混调器 | 0 | F0 |
| **D1** | 电子天平 1 | 1 | F3 |
| **D2** | 电子天平 2 | 2 | F3 |
| **D3** | PH 计 | 3 | F3 |
| **D4** | 水质检测仪 | 4 | F5 |
| **D5** | 表界面张力仪 | 5 | F5 |
| **D6** | 搅拌器 | 6 | F7 |
| **D7** | 水浴锅 | 7 | F7 |
| **D8** | 粘度计 | 8 | F8 |

## 系统架构

- **YOLO 检测**：在全景图中精确定位 D0~D8 各类仪表。
- **自动裁剪**：根据检测到的 bbox 自动抠出特写。
- **多模态识别**：调用 LLM 对特写图进行结构化识别。

## 目录结构

```
ocr-new/

│   ├── api/                  # API 路由
│   ├── models/               # 数据库模型
│   └── services/             # 核心服务 (YOLO, LLM, Pipeline)
├── frontend/                 # 前端应用 (Next.js)
├── config.py                 # 基础配置
├── instrument_reader.py      # 核心读取逻辑
├── camera_images/            # 物理相机图片缓存 (F0-F8)
└── docs/                     # 系统文档
```
